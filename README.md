# 🚆 IRCTC Rail Connect – UI/UX Case Study

> A UI/UX redesign case study of the IRCTC Rail Connect mobile application focused on improving the train ticket booking experience through structured UX methodologies and interface design.

---

## 📌 Project Overview

This project redesigns the booking experience of the IRCTC Rail Connect application using core UI/UX design principles.

The case study focuses on:
- Improving usability
- Simplifying navigation
- Reducing cognitive load
- Creating a consistent booking experience

---

## 🧠 UX Process

The project was divided into three major phases:

### 1️⃣ Information Architecture (IA)

Information Architecture defines how information, screens, and navigation are organized inside the application.

The IA focuses on:
- User flow structure
- Navigation hierarchy
- Booking journey organization
- Task-based screen flow

The architecture covers the complete booking journey:
- Login / Register
- Train Search
- Passenger Details
- Payment
- Booking Confirmation

---

### 2️⃣ Design System

The Design System creates a consistent visual language for the application.

It includes:
- Typography system
- Color palette
- Buttons
- Input fields
- Cards
- Navigation components
- Spacing system
- Iconography

The system helps maintain:
- Visual consistency
- Reusable UI components
- Better usability
- Scalable interface design

---

### 3️⃣ Mid-Fidelity Wireframes

Mid-Fidelity Wireframes are structured interface layouts that focus on usability and screen functionality instead of final visual styling.

These wireframes were designed to:
- Visualize user interaction flow
- Organize content hierarchy
- Test navigation structure
- Represent the booking experience clearly

Included Screens:
- Login
- Register
- Home Dashboard
- Search Train
- Train Listing
- Passenger Details
- Payment
- Booking Confirmation
